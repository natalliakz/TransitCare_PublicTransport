# TransitCare Fleet Maintenance Demo

An R-based demo project showcasing predictive maintenance analytics for public transport fleet management, featuring Shiny applications with usage telemetry powered by [shiny.telemetry](https://appsilon.github.io/shiny.telemetry/).

> **Note**: This project uses synthetic data generated for demonstration purposes. All company names, vehicle identifiers, and metrics are fictional.

## Overview

TransitCare Fleet Solutions is a fictional public transport company used to demonstrate:

- **Predictive Maintenance Analytics**: ML-powered failure risk prediction
- **Interactive Dashboards**: Real-time fleet health monitoring with Shiny
- **Usage Analytics**: Application telemetry with shiny.telemetry and SQLite storage
- **REST APIs**: Programmatic access to fleet data and predictions with Plumber

## Quick Start

### 1. Install Dependencies

```r
# Install renv if not already installed
install.packages("renv")

# Restore project dependencies
renv::restore()
```

### 2. Generate Synthetic Data

```r
source("data/generate_data.R")
```

### 3. Train the Model

```r
source("ml/train_model.R")
```

### 4. Run Applications

**Main Dashboard:**
```r
shiny::runApp("app.R")
```

**Telemetry Dashboard:**
```r
shiny::runApp("telemetry_dashboard.R")
```

**REST API:**
```r
plumber::plumb("api.R")$run(port = 8000)
```

**Render EDA Report:**
```bash
quarto render eda.qmd
```

## Features

### Main Dashboard (app.R)

- Fleet health overview with key metrics
- Vehicle health distribution and maintenance analysis
- Individual vehicle details with sensor readings
- ML-powered failure risk predictions
- **Built-in telemetry tracking** via shiny.telemetry

### Telemetry Dashboard (telemetry_dashboard.R)

- Session tracking and user behavior analytics
- Navigation patterns and input interactions
- Real-time activity monitoring
- Demonstrates multi-app architecture on Posit Connect

### REST API (api.R)

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/health` | GET | API health check |
| `/fleet/summary` | GET | Fleet statistics |
| `/vehicles` | GET | List vehicles (with filters) |
| `/vehicles/health` | GET | Vehicle health data |
| `/vehicles/<id>` | GET | Specific vehicle details |
| `/maintenance` | GET | Maintenance records |
| `/predict` | POST | Failure risk prediction |
| `/model-info` | GET | Model metadata |

## Customization

### Branding

Edit `_brand.yml` to customize colors, typography, and company information. The Shiny apps automatically apply these settings via bslib.

### Data

Modify `data/generate_data.R` to adjust:
- Fleet size and composition
- Maintenance patterns
- Vehicle health distributions
- Sensor reading ranges

### Model

The Random Forest classifier in `ml/train_model.R` can be tuned or replaced with alternative algorithms from the tidymodels ecosystem.

## Deployment to Posit Connect

Both Shiny applications and the API can be deployed to Posit Connect:

```r
# Deploy main app
rsconnect::deployApp("app.R", appName = "transitcare-maintenance")

# Deploy telemetry dashboard
rsconnect::deployApp("telemetry_dashboard.R", appName = "transitcare-telemetry")

# Deploy API
rsconnect::deployAPI("api.R", appName = "transitcare-api")
```

**Important**: For shared telemetry data between apps on Connect, configure a shared SQLite database path or migrate to PostgreSQL for production use.

## About shiny.telemetry

[shiny.telemetry](https://appsilon.github.io/shiny.telemetry/) by Appsilon enables:

- Session and navigation tracking
- Input interaction logging
- Custom event logging
- Multiple storage backends (SQLite, PostgreSQL, MariaDB)

This demo uses SQLite for simplicity, storing telemetry in `telemetry.sqlite`.
